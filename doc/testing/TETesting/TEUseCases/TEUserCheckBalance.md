### TEUserCheckBalanceTest

#### Actors
* Tester / User

#### Description
* Tester makes sure that user can check balance

#### Preconditions
* User has created an account
* User has logged in

#### Postconditions
* Successful - Tester has checked balance
* Unsuccessful – Tester could not check balance

#### Dialog
1. User opens Application
2. User is greeted with the login screen with option to create account.
3. User creates account/logs in to application 
4. User checks balance
